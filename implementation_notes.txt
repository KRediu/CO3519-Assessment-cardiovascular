PRESENT IMPLEMENTATION
-------------------
Dataset: cardio_train.csv, binary target cardio (0/1). One training script per model, each writes its own metrics CSV; then a comparison script, evaluation figures, and an optional HGB tuning round.

1_eda.py: raw vs cleaned counts → results/metrics/1_eda_summary.csv

2_preprocessing.py: clean(), features, stratified split, ColumnTransformer; dense matrices for HGB/np.save → data/processed/*.npy, feature_names.csv, results/metrics/2_split_summary.csv

3_random_forest.py → results/metrics/3_random_forest_metrics.csv
4_logistic_regression.py → results/metrics/4_logistic_regression_metrics.csv
5_mlp.py → results/metrics/5_mlp_metrics.csv
6_hist_gradient_boosting.py → results/metrics/6_hist_gradient_boosting_metrics.csv

7_results_comparison.py: concatenates the above (and 9_tuned_model_metrics.csv if present) → results/metrics/7_model_comparison.csv

8_evaluation_plots.py: ROC, PR, confusion matrices (estimators defined inline to match scripts 3, 4, 5, 6) → results/figures/*.png

9_tuning_round2.py: default HGB (same as script 6) vs RandomizedSearchCV; results/metrics/9_tuning_round2.csv and 9_tuned_model_metrics.csv (for 7_)

utils.py: paths, load/clean/make_xy/preprocessor, save_np, save_metrics_csv.

Typical run order: 1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9 → 7 (optional, to refresh comparison with tuned row).
