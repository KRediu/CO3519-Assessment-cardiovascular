PRESENT IMPLEMENTATION
-------------------
Dataset: cardio_train.csv, binary target cardio (0/1).

1_eda.py: full audit csv, raw vs cleaned, duplicate checks, bad blood-pressure count, class counts.

2_preprocessing.py: cleaning, feature engineering, stratified split, preprocessing pipeline, processed arrays.

3_umap_visualization.py: UMAP projection on a stratified sample, plus embedding csv.

4_random_forest.py: round-1 RF metrics csv.
5_logistic_regression.py: round-1 logistic metrics csv.
6_mlp.py: round-1 MLP metrics csv.
7_hist_gradient_boosting.py: round-1 HGB metrics csv.

8_hyperparameter_tuning.py: RandomizedSearchCV over LR, RF, MLP, HGB, saves tuned metrics and best-params json.

9_results_comparison.py: combines round-1 model csvs and tuned csv (if present) into 9_model_comparison.csv.

10_round_delta_report.py: compares round_1 vs round_2_tuned by model and writes delta metrics.

11_visualizations.py: class balance, correlation heatmap, ROC, PR, confusion matrices.

run_pipeline.py: full orchestrator, clears generated outputs first, runs numbered scripts in order, optional skip flags.

utils.py: shared paths, data load/clean/audit/feature engineering, preprocessor builder, metrics saver.

notes for report
------------------
We used cardio_train.csv as a binary risk task (cardio 0/1), then cleaned obvious bad rows (invalid blood pressure ordering/values, extreme height/weight outliers) before modeling.
We engineered age_years and bmi, dropped id, then applied a standard preprocessing pipeline (imputation + scaling / encoding) with a stratified train-test split.
Round 1 used four models: random forest, logistic regression, MLP, and histogram gradient boosting; each was evaluated with CV plus hold-out test metrics.
We generated comparison outputs and figures so results are not just one score: ROC/PR behavior, confusion patterns, class balance, and correlation view.
Round 2 tuning used RandomizedSearchCV across all four models, then we compared tuned vs untuned in the comparison + delta outputs.
Main result pattern: gains from tuning were generally small, some models improved marginally while others were flat/slightly down, which suggests model overlap + limited headroom from parameter-only changes.
The strongest practical takeaway is consistency across models around similar performance bands, rather than one model massively dominating.
The useful narrative is: solid base pipeline, sensible cleaning/feature work, fair model comparison, then tuning + delta analysis to show what changed and by how much.
Limits to mention: tabular-only features, no external data enrichment, and improvements mostly incremental after first-pass modeling.
